# AI Conversation Tree Agent Guide

## 项目目标

AI 对话树是一个本地优先的浏览器扩展，用来在 ChatGPT 和 Gemini 页面右侧展示可搜索、可折叠、可拖拽重排、可导出的对话树。长期目标是把长对话整理成可靠的结构化导航，让用户能快速回到问题、理解上下文关系，并导出可复用的对话资料。

这个项目会长期运营。用户会持续通过对话提出新功能、体验优化和 bug 修复请求。每次开始工作时先读取本文件，再按当前请求和仓库现状行动。

## 长期产品原则

- 保持纯本地实现，不接入外部 AI API，不新增远程服务或第三方追踪。
- 不调用 ChatGPT 或 Gemini 的私有接口；基于当前页面 DOM 和可见文本分析对话结构。
- 优先保护用户隐私：调试日志默认脱敏，不保存完整回答到持久化状态，避免把敏感对话内容写入仓库或远端。
- 优先保证长对话可用性：扫描稳定、跳转准确、渲染流畅、状态可恢复、导出可靠。
- 改动 DOM 扫描、跳转、导出、删除、撤销、拖拽、排序或持久化时，要同时考虑 ChatGPT 和 Gemini 两个平台。
- 任何新增站点支持都必须同步检查 `manifest.json` 权限、README、隐私说明、扫描逻辑、测试和降级表现。

## 当前技术结构

- `manifest.json`：Chrome/Edge MV3 扩展配置、权限、注入脚本顺序。
- `content-core.js`：内容脚本共享工具，包括调试日志、状态序列化、活跃节点标记等。
- `hard-algorithm.js`：本地硬算法，负责提取问题、压缩回答匹配文本、推断父子关系。
- `content.js`：主内容脚本，负责站点识别、DOM 扫描、面板渲染、搜索、跳转、拖拽、删除、撤销、排序、导出和持久化。
- `content.css`：页面内对话树面板样式。
- `popup.html` / `popup.js` / `popup.css`：扩展按钮弹窗，用于查看/恢复当前对话状态并复制脱敏调试日志。
- `tests/performance-optimizations.test.js`：Node 回归测试，覆盖长对话性能、缓存、状态瘦身、调试日志脱敏和扫描性能约束。
- `dist/`：发布产物目录，除非用户要求打包发布，否则不要随手重建或改动。

## 开发方式

- 这个仓库没有构建流程，源码直接作为浏览器扩展加载。
- 优先保持原生 JavaScript、CSS、HTML 的现有风格，不引入框架或打包工具，除非用户明确要求并且收益足够大。
- 搜索和阅读代码优先使用 `rg` / `rg --files`。
- 修改代码前先看相关函数周围上下文，尤其是 `content.js` 的扫描、状态和渲染路径。
- 对共享行为的修改要尽量补充或更新 `tests/performance-optimizations.test.js`。
- 文案和界面默认使用中文，保持简洁，避免在页面里加入解释性教程文字。

## 验证要求

- 文档改动至少检查 `git diff --check`。
- JavaScript 逻辑改动优先运行：

```sh
node tests/performance-optimizations.test.js
```

- 如果修改了扩展注入、权限、弹窗或跨文件加载顺序，手动提醒用户在浏览器扩展管理页重新加载扩展，并在 ChatGPT/Gemini 页面验证。
- 如果测试无法运行，要明确说明原因，不要声称已通过。

## Git 协作规则

- 用户希望每次修改后都提交并推送。默认完成一次代码或文档修改后，执行验证、提交、推送，除非用户明确说不要提交或不要推送。
- 提交前必须运行 `git status -sb` 和查看相关 diff，确认提交范围。
- 工作区可能已有用户未提交改动。不要回滚、覆盖或静默暂存这些改动；只暂存本次任务相关文件。如果需要把用户已有改动一起提交，必须先得到用户明确同意。
- commit message 用简短中文，说明本次实际改动。
- 默认推送当前分支到 `origin`。如果推送失败，报告失败原因和已完成的本地提交哈希。
