# 暖桌面手账 UI 与主题切换设计

## 背景

当前 AI 对话树面板偏深色工具风，用户希望改成更可爱的视觉风格，并同时支持暗黑模式、浅色模式，以及根据浏览器系统设置自动切换。经过可视化预览，选定方向为“暖桌面手账”：奶白、浅绿、木色调，温柔耐看，适合长时间停靠在 ChatGPT/Gemini 页面右侧。

## 目标

- 将页面内对话树面板改成“暖桌面手账”风格。
- 将扩展弹窗同步改成同一套浅色/暗色视觉系统。
- 支持主题模式：`跟随系统`、`浅色`、`暗色`。
- 默认使用 `跟随系统`，用户手动选择后持久化到本地。
- 保持现有功能不变：扫描、搜索、拖拽、删除、撤销、排序、导出、关闭/恢复、调试日志。

## 非目标

- 不引入构建流程、前端框架、远程字体或图片资源。
- 不修改 DOM 扫描、树关系推断、导出内容语义。
- 不扩大站点权限。
- 不重新设计整个交互信息架构，只做视觉系统和主题控制。

## 视觉方向

浅色模式使用奶白和浅绿为底色，辅以木色/琥珀强调。整体应像一张轻量手账贴在页面右侧：柔和、清晰、亲近，但不使用装饰图片或过度拟物元素。

暗色模式使用深绿灰作为底色，文字保持米白/浅绿，高亮使用柔和琥珀与嫩绿色。暗色不能只是浅色取反，要保证在 ChatGPT/Gemini 暗色页面上低干扰、可读。

关键视觉原则：

- 圆角更柔和，但控件仍保持清楚的边界和点击感。
- 按钮、输入框、选择框都使用同一组主题变量。
- 当前节点、搜索命中、拖拽目标、危险删除状态必须有足够区分度。
- 文字尺寸和密度保持工具属性，不做大幅营销式排版。

## 主题控制

新增主题偏好值：

- `system`：跟随 `prefers-color-scheme`。
- `light`：强制浅色。
- `dark`：强制暗色。

默认值为 `system`。用户选择后保存到本地浏览器存储。实际生效主题通过以下规则计算：

1. 如果偏好为 `light` 或 `dark`，直接使用该模式。
2. 如果偏好为 `system`，读取 `window.matchMedia("(prefers-color-scheme: dark)")`。
3. 系统主题变化时，只有偏好为 `system` 才自动刷新面板主题。

主题偏好应与对话树结构状态分开保存，避免每个对话重复存一份。建议使用独立 localStorage key，例如 `cgpt_tree_theme_preference_v1`。

## 面板实现

`content.js` 负责：

- 在状态中新增 `themePreference` 和当前 resolved theme。
- 创建主题切换控件，放在面板顶部操作区，文案为 `系统` / `浅色` / `暗色`。
- 初始化时读取主题偏好并设置面板 class/data attribute。
- 监听主题切换控件变化，保存偏好并立即更新面板。
- 当系统颜色变化时，如果当前偏好为 `system`，刷新 resolved theme。

`content.css` 负责：

- 使用 CSS 变量定义浅色和暗色两套 token。
- 通过 `#cgpt-tree-panel[data-theme="light"]` 和 `#cgpt-tree-panel[data-theme="dark"]` 应用主题。
- 保持 SVG 树节点、折叠按钮、拖拽 ghost、hover tooltip、页面高亮样式同步主题。
- 保持导出 SVG 的颜色可读；导出可以继续使用固定浅/暗中性配色，除非实现时成本很低。

## 弹窗实现

`popup.css` 使用同一套“暖桌面手账”视觉语言，并通过 `prefers-color-scheme` 自动适配浅/暗色。

弹窗不一定需要放主题切换控件，第一版以跟随系统为主；如果内容脚本的主题偏好需要在弹窗中可改，再扩展 `popup.js` 读取/写入同一个 localStorage key。为了降低实现范围，首版优先在页面内面板提供手动切换。

## 持久化与兼容

- 主题偏好独立保存，不写入每个对话的树状态。
- 旧用户没有主题偏好时自动使用 `system`。
- 旧树状态不需要迁移。
- 如果 localStorage 写入失败，面板仍然可以在当前页面临时切换主题，但刷新后可能回到默认。

## 测试与验证

自动验证：

```sh
node tests/performance-optimizations.test.js
git diff --check
```

手动验证：

- 在浅色浏览器/系统主题下打开 ChatGPT 页面，面板默认浅色。
- 在暗色浏览器/系统主题下打开 ChatGPT 页面，面板默认暗色。
- 手动切到浅色，刷新页面后仍为浅色。
- 手动切到暗色，刷新页面后仍为暗色。
- 切回系统后，跟随浏览器主题。
- Gemini 页面也能显示同样主题控件和视觉样式。
- 搜索、折叠、拖拽、删除、撤销、线性排序、导出按钮仍可用。

## 风险

- `content.js` 已经较大，主题逻辑应保持小而集中，避免搅动扫描和树算法。
- 主题控件放在顶部可能挤压现有按钮，需要注意窄屏和折叠状态。
- 深色和浅色 CSS token 需要覆盖 SVG 文本、节点、滚动条、tooltip 和当前页面高亮，否则会出现局部旧风格。
